# 푸시 세그먼트 운영 대시보드 — 이용 가이드

---

## 1. 개요 (Executive Summary)

이 대시보드는 **"같은 푸시를 전체 발송하지 말고, 반응할 사람에게만 보내자"**를 위한 도구입니다.

발송 가능한 전체 유저를 행동 패턴(시청 시간대, 구매, 활동량 등)에 따라 **10개 세그먼트(S1~S10)**로 나눠두고, 푸시를 보낼 때 **잘 반응할 세그먼트부터 골라** 발송 명단(CSV)을 만들어 줍니다. 그 결과 **발송량은 줄이면서 오픈은 대부분 지키는** 효율적 발송이 가능합니다.

| 화면(탭) | 한 줄 설명 |
| --- | --- |
| **① 푸시 현황** | 보낸 푸시 성적표 + **세그먼트별 전환·커버리지** 분석 |
| **② 유저 세그먼트** | 전체 유저를 10개 세그먼트(S1~S10)로 나눈 현재 상태 확인·갱신 |
| **③ 푸시 분배** | 보낼 푸시를 입력하면 → 세그먼트별 배정 → 발송 명단 CSV 다운로드 |

> **세그먼트 번호(S1~S10)는 발송 우선순위 순서**입니다. `S1`이 가장 잘 반응하는 그룹(헤비구매 VIP), `S10`이 가장 낮은 그룹(휴면). 모든 화면에서 같은 번호를 씁니다.

## 3. 화면별 사용법

### 3.1 ① 푸시 현황

상단에 **오픈율 추이**와 **카테고리별 평균 오픈율** 그래프가 있고, 그 아래로 네 가지 섹션이 화면 순서대로 있습니다: **A. 세그먼트별 전환 → B. 푸시 방송 매칭·결과 추가 → C. 푸시 메시지 성과 랭킹 → D. 세그먼트별 커버리지**.

#### A. 세그먼트별 전환
각 푸시를 **오픈한 유저**가 그 방송을 **유효시청·구매까지 갔는지**를 세그먼트(S1~S10)별로 보여줍니다.

- **차트**: 세그먼트별 시청전환율·구매전환율 (전체 평균은 점선으로 표시).
- **표**: 세그먼트 | 오픈 | 유효시청(명·%) | 구매(명·%) | **객단가**. 전환율은 **오픈자 기준**(오픈한 사람 중 시청/구매한 비율), 객단가는 **전환 구매 GMV ÷ 구매자 수**(구매자 1인당).
- **[집계 실행 / 갱신] 버튼**으로 계산합니다(수 분 소요). 캐시된 결과를 보여주므로, 아래 경우엔 다시 눌러야 반영됩니다:
  - 새 푸시의 방송 매핑을 추가했을 때 · 유저 세그먼트를 최신화했을 때 · 최신 수치를 보고 싶을 때
- ※ push→방송(content) 연결은 **운영자가 수기 확정한 매핑** 기준입니다(자동 추정이 아님).

#### B. 푸시 방송 매칭·결과 추가
특정 푸시의 방송을 자동으로 찾아 전환을 미리보고 커버리지에 추가합니다.

- **push_seq · 판매자이름 · 카테고리** 3개를 입력하면, 발송시각 전후의 그 판매자 방송을 자동으로 찾아(discover) **오픈 → 유효시청 → 구매**를 미리 보여줍니다.
- ※ 판매자이름은 띄어쓰기·대소문자·이모지까지 **정확히 일치**해야 합니다(예: `SABABA 사바바`).
- **[커버리지에 추가]**를 누르면 push→방송 매핑이 저장됩니다. 이후 **[집계 갱신]**을 눌러야 위 'A. 세그먼트별 전환'·아래 'D. 커버리지'에 반영됩니다(자동 저장·자동 반영 아님).

> content_seq를 쉼표로 직접 입력하는 단건 전환 조회는 '③ 푸시 분배' 탭의 **푸시알림 효과검증(A/B)** 섹션에 있습니다.

#### C. 푸시 메시지 성과 랭킹 (TOP / BOTTOM 10)
등록된 푸시를 **메시지 단위로 줄 세웁니다**.

- **정렬 기준** 선택: 거래전환율 · 유효시청률 · 객단가 · 거래수.
- **최소 오픈수** 필터: 표본이 작으면 비율이 출렁이므로 그 이하 푸시는 제외합니다.
- **[집계 갱신]** 시 위 전환 집계와 함께 최신화됩니다.

#### D. 푸시 메세지 세그먼트별 커버리지
모든 푸시를 **세그먼트별로 펼친 표**입니다. 각 푸시가 어느 세그먼트에 얼마나 발송됐고 어떻게 전환됐는지 한눈에 비교할 수 있습니다(푸시당 S1~S10 + 미배정 = 11행).

- **푸시 정보(좌측)**: push 번호 · 판매자 · 제목 · 카테고리(색 칩) · 발송시각 · 전체 발송/도달/오픈
- **세그먼트별(우측)**: 세그먼트 | 발송 | 도달 | 오픈 | 유효시청률 | 거래전환율
- **괄호 안 누적비율**: 발송·도달·오픈 옆의 `(%)`는 S1부터 내려가며 쌓인 누적 비율로, 맨 아래(미배정)에서 100%가 됩니다.
- **굵은 인디고 선**: 오픈자 누적이 **50%를 넘는 지점** — 위쪽 세그먼트들이 오픈자 절반을 커버한다는 뜻(반응 집중도).
- **검색**: 제목·판매자·카테고리·푸시번호로 필터. **발송시각** 머리글을 클릭하면 정렬됩니다.
- push 번호를 클릭하면 위 **B. 푸시 방송 매칭**에 자동 입력됩니다.

> 표의 **발송**은 그 푸시가 실제로 보낸 인원(`push_send_history`)입니다. 분배 탭의 "전체발송"(전체 모수 ~41만)과 다른데, 이는 마케팅이 푸시마다 일부에게만 보내고 특히 휴면(S10, 전체의 70%)엔 거의 안 보내기 때문입니다.

### 3.2 ② 유저 세그먼트

발송 가능한 전체 유저를 행동 패턴에 따라 **10개 세그먼트로 나눈 결과를 한 표**로 보여줍니다.

- 상단 칩(`● 적용 중` · 총 인원 · 기준 시점 · 최근 N일 시청자)이 **현재 분배에 쓰이는 데이터 기준**입니다.
- **세그먼트 이름**: `S1 VIP 코어` ~ `S10 휴면` (발송 우선순위 순). 작은 회색 글씨 `원본 C9`는 분석 모델 원본 라벨로, 노트북·과거 자료 대조용 참고값입니다.
- **Tier(신호등)** — 발송 액션 가이드:
  - 🟢 **상시 발송** (S1~S3): 반응이 가장 좋은 핵심 그룹
  - 🟡 **조건부 발송** (S4~S5): 카테고리가 맞을 때 발송
  - 🔴 **발송 제외 / 리액티베이션** (S6~S10): 평소 제외, 재활성화 캠페인에만
- 그 외 컬럼: 인원·비중 · 5월 실측 CTR · lift(평균 대비 배율) · 특징(행동 요약·친절 설명)

**[유저 세그먼트 최신화하기]**: 최근 N일 시청자(기본 180일) 기준으로 세그먼트를 다시 배정합니다(약 3~15분, 페이지를 닫아도 계속). 기준 시점이 7일을 넘으면 빨간 경고가 뜹니다.

> 💡 세그먼트를 최신화하면 세그먼트 구성이 바뀌므로, **'세그먼트별 전환·커버리지'도 [집계 갱신]**을 다시 눌러야 일치합니다.

### 3.3 ③ 푸시 분배 (핵심 기능)

보낼 푸시를 입력하면 어느 세그먼트에 어떤 푸시를 보낼지 자동 배정하고 **푸시별 발송 대상 명단(CSV)**을 만들어 줍니다.

**사용 순서**

1. **푸시 입력**: 제목(메모용) + 카테고리 + 발송 시간대(오전/오후/저녁, 무관 가능). [+ 푸시 추가]로 여러 개 동시 입력 가능.
2. **발송 대상 모드 선택**: 상단 `발송 대상:` 토글에서 **자동(상위 N개)** 또는 **수동(그룹 직접 선택)**. 수동은 각 푸시마다 세그먼트(S1~S10)를 칩으로 직접 고릅니다.
3. **발송 풀 슬라이더** *(자동 모드에서만 노출)*: 상위 몇 개 세그먼트까지 보낼지 선택(기본 5개 = 🟢🟡 등급). 좁힐수록 발송량↓ 효율↑. *수동 모드에서는 보이지 않습니다.*
4. **(선택) A/B 대조군 %**: 0~50% 입력 시, 실행 후 푸시별로 **발송 CSV**와 **대조군(미발송) CSV**가 따로 생성됩니다.
5. **실행** → 분배 매트릭스(세그먼트×푸시 배정) · 푸시별 요약 · **[CSV 다운로드]**(발송 대상 user_id).
6. 다운로드한 CSV를 푸시 발송 시스템에 업로드해 발송.

> **푸시알림 효과검증 (A/B)** — 페이지 하단 상시 표시 섹션. 발송 며칠 뒤 **발송 CSV·대조군 CSV**를 올리고 **push_seq·content_seq**를 입력하면, 발송군 vs 대조군의 유효시청·구매·GMV 증분과 유의성(z검정)을 계산합니다.

**효율 지표**

| 지표 | 의미 |
| --- | --- |
| 발송 절감 | 전체 발송 대비 몇 %를 아꼈는지 |
| 클릭 커버리지 | 전체 발송 시 얻을 클릭 중 몇 %를 지켰는지 |
| 오픈율 배율 | 전체 발송 대비 오픈율이 몇 배 좋아지는지 |

> 기대 오픈·오픈율은 **발송 대비** 기준입니다(모델 CTR은 도달 대비라, 발송 대비로 환산할 때 도달률을 반영). "전체발송 시"는 스냅샷 전체 모수(~41만)에 다 보낸 가정으로, 비교 기준값입니다.


## 5. 주의사항

- 🔒 **다운로드한 CSV는 유저 단위 개인 데이터입니다. 외부 공유·메신저 전달 금지.**
- 📅 분배 명단은 **만든 시점의 스냅샷 기준**입니다. 7일이 지나면 경고가 뜨며, 발송 전 최신화를 권장합니다.
- 🔗 **세그먼트별 전환·커버리지는 수기 매핑 기반**입니다. 새로 보낸 푸시는 방송 매핑을 추가한 뒤 [집계 갱신]을 눌러야 나타납니다. 발송/도달 수치는 갱신 시점 기준이라, 이후 이벤트가 더 쌓이면 조금 늘 수 있습니다.
- 📊 화면의 예상 오픈·효율 수치는 **2026년 5월 데이터로 학습한 모델 기준 추정치**입니다. 실제 효과는 일부 무작위 그룹과 비교(A/B)로 확인하는 것이 정확합니다.
- 💾 분배 결과 다운로드가 갑자기 안 되면(오류 화면) **분배를 다시 실행**하면 됩니다.

## 6. 자주 묻는 질문 (FAQ)

**Q. 최신화·집계 도중 브라우저를 닫아도 되나요?**
네. 작업은 서버에서 계속 진행됩니다. 나중에 다시 들어와 결과를 확인하세요.

**Q. 새로 보낸 푸시가 '세그먼트별 전환'에 안 보여요.**
그 푸시의 방송 매핑이 아직 없어서입니다. 매핑을 추가한 뒤 [집계 갱신]을 누르면 반영됩니다(담당자(데이터팀) 문의).

**Q. 커버리지의 발송 합(~10만)이 분배의 전체발송(~41만)과 왜 다른가요?**
세는 모집단이 다릅니다. 커버리지 발송은 **그 푸시가 실제 보낸 인원**(과거 실측, 일부 타겟)이고, 분배의 전체발송은 **발송 가능한 전체 모수**입니다. 특히 전체의 70%가 휴면(S10)인데 실제론 거의 안 보내므로 차이가 큽니다.

**Q. 분배 발송 인원이 전체 유저보다 한참 적은데 정상인가요?**
정상입니다. 반응이 낮은 세그먼트(🔴 등)를 제외하고 상위 그룹에만 보내는 것이 이 도구의 목적입니다. 발송 풀 슬라이더로 범위를 조절하세요.

**Q. 같은 조건으로 다시 실행하면 명단이 달라지나요?**
아니요. 같은 스냅샷·같은 입력이면 같은 결과가 나오도록 고정되어 있습니다.

**Q. "백엔드에 연결할 수 없습니다"가 떠요.**
서버가 꺼진 상태입니다. 담당자(데이터팀)에게 재기동을 요청하세요.

## 7. 용어 사전

| 용어 | 쉬운 설명 |
| --- | --- |
| 세그먼트 (S1~S10) | 행동 패턴이 비슷한 유저 그룹. 번호는 발송 우선순위(S1=최우수, S10=휴면). `원본 C#`은 분석 모델 라벨 |
| 미배정 | 현재 스냅샷에 없는(분류 밖) 유저 — 발송엔 포함되나 세그먼트엔 안 들어감 |
| Tier (신호등) | 발송 액션 등급: 🟢 상시 발송 / 🟡 조건부 발송 / 🔴 발송 제외·리액티베이션 |
| 발송 / 도달 / 오픈 | 보낸 수 / 휴대폰에 실제 도착한 수 / 유저가 푸시를 연 수 |

| 유효시청 / 유효시청률 | 기준 시간(10초) 이상 실제 시청 / 오픈자 중 유효시청 비율 |
